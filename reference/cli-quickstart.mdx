## Step 1: Install the CLI

Install the SuprSend CLI using your preferred method:

<Tabs>
<Tab title="Homebrew (Recommended)">
**For macOS or Linux users**
```bash
brew tap suprsend/tap
brew install --cask suprsend
```
</Tab>
<Tab title="Binary Releases">
**Best for Windows users, servers or CI/CD environments**

Download the binary from [GitHub Releases](https://github.com/suprsend/cli/releases) and then install it using:
```bash
# Linux/macOS
chmod +x suprsend
sudo mv suprsend /usr/local/bin/

# Windows: Extract suprsend.exe to your PATH
```
</Tab>
<Tab title="Source Build">
**Best for project contributors** or advanced users who want to always have access to the latest binary.
Requires Go (version 1.20 or later)
```bash
# 1. Clone the repository
git clone https://github.com/suprsend/cli.git
cd cli/cmd/suprsend

# 2. Build the binary
go build -o suprsend

# 3. Move the binary to your PATH for easy access
sudo mv suprsend /usr/local/bin/
```

</Tab>
</Tabs>

## Step 2: Authentication

Get your service token from [SuprSend Dashboard â†’ **Account Settings** â†’ **Service Tokens**](https://app.suprsend.com/en/account-settings/service-tokens).

There are three ways to authenticate the CLI: 
- Setting environment variable
- Using profiles
- Passing service token as a command flag

We generally recommend setting up profiles to ensure that service token is not exposed in the shell history but if you're setting up CLI to start using MCP server, then you can use environment variable or command flag.

**Setting up environment variable:**
```bash
export SUPRSEND_SERVICE_TOKEN="your_service_token_here"
```
**Using profiles:**
```bash
suprsend profile add --name "my-workspace" --service-token "your_service_token_here"
suprsend profile use --name "my-workspace"
```

**As command flag when starting MCP server:**
```bash
suprsend start-mcp-server --service-token "your_service_token_here"
```

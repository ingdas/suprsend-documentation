Push workflows from local to suprsend. Pushed translations are saved in draft version. Make sure to pass `--commit=true` flag to commit the translations while pushing, or use the [`translation commit`](/reference/cli-translation-commit) command separately.

## Syntax

```bash
suprsend translation push [flags]
```

## Flags

| Flag | Description | Default |
|------|-------------|---------|
| `-h, --help` | Show help for the command | – |
| `-c, --commit string` | Commit the translation (--commit=true) | `false` |
| `-m, --commit-message string` | Commit message for the translation | – |
| `-d, --dir string` | Directory for translations pull to | `./suprsend/translation` |

## Example

```bash
# Push translations from default directory
suprsend translation push

# Push translations from custom directory
suprsend translation push --dir dev-environment/translations

# Push translations and commit them
suprsend translation push --commit=true --commit-message "Updated French translations"

# Push translations to production workspace
suprsend translation push --workspace production
```
